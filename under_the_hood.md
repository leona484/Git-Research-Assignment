 (Under the Hood) در Fork و Clone

1- هنگام Fork چه اتفاقی می‌افتد؟

وقتی یک مخزن را Fork می‌کنید، GitHub یک کپی کامل از مخزن اصلی را روی حساب کاربری شما ایجاد می‌کند.

در این فرآیند:
- تمام فایل‌ها، commitها و branchها کپی می‌شوند
- مخزن جدید مالکیت جداگانه دارد
- ارتباط با مخزن اصلی حفظ می‌شود تا بتوان Pull Request ارسال کرد
- تغییرات شما مستقیماً روی پروژه اصلی اعمال نمی‌شود

Fork فقط روی سرور GitHub انجام می‌شود و هیچ فایلی روی سیستم شما دانلود نمی‌شود.



2- هنگام Clone چه اتفاقی می‌افتد؟

وقتی یک مخزن را Clone می‌کنید، Git یک نسخه کامل از مخزن را روی سیستم محلی شما دانلود می‌کند.

در این فرآیند:
- تمام فایل‌ها و تاریخچه commitها دریافت می‌شود
- یک پوشه .git ساخته می‌شود که اطلاعات Git را نگه می‌دارد
- ارتباط با مخزن آنلاین به عنوان remote ذخیره می‌شود
- امکان commit، pull و push از طریق خط فرمان یا GitHub Desktop فراهم می‌شود

Clone باعث می‌شود بتوانید به صورت آفلاین روی پروژه کار کنید.
